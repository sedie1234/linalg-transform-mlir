구조적 발견 — pass 내부 구성의 두 가지 형태

  • greedy pattern pass (8/10) — runOnOperation → populate*Patterns → pattern.matchAndRewrite → 핵심 함수 → applyPatternsAndFoldGreedily. pattern은 핵심 함수의 얇은 어댑터이고, 이식 시 populate 함수 하나(또는 핵심 함수 직접 호출)만 가져오면 된다.
  • dialect-conversion pass (2/10) — convert-elementwise-to-linalg(ConversionTarget + applyPartialConversion)와 linalg-detensorize(TypeConverter + cost-model + applyFullConversion). 합법성(legal/illegal op 집합)과 타입 변환이 개입하는 형태로, 이식 시 target 구성까지 함께 가져와야 한다.
  • 한 기계 여러 pass — lowering 3종(convert-linalg-to-{loops,affine-loops,parallel-loops})은 Loops.cpp의 한 template(linalgOpToLoopsImpl<LoopTy>) instantiation 차이일 뿐. indexing_maps→load/store index, payload→loop body 변환이 공통 본체.
