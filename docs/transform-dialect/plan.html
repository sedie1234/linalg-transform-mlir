P2 — Transform Dialect ("외부 schedule")

linalg(P1, cycle 0001~0010) 완료 후속. 변환을 IR(데이터)로 컴파일러 밖에서 정의·적용하는 transform dialect를 사용법·효과 중심으로 학습하고, 핵심 의문("외부에서 변환규칙을 정의·적용하는 이점")을 실측으로 못박는다. 개념 깊이 파기는 0011-transform-tile-first.html 참고.

방법론 (P1 루프 유지 + usage-first)

각 work = schedule 작성 → mlir-opt -transform-interpreter 적용 → IR 전후 관찰/측정 → 5섹션 log → 누적 시 NAS 지식 전이 + Plane Page. P1과 달리 "C++ 해부"가 아니라 "schedule을 직접 쓰고 효과를 관찰"이 주축(의문이 user-facing이므로); 내부구조 해부는 Phase D로 뒤에. payload는 익숙한 linalg op 재사용. interpreter는 in-tree mlir-opt·out-of-tree my-mlir-opt 둘 다 -transform-interpreter 지원. 무거운 빌드는 T14(커스텀 op)뿐.

워크스페이스 대조축

같은 변환(tile/fuse/pack/vectorize/generalize)을 세 패러다임으로 1:1 대조 — ① affine(polyhedral) 완전 자동(ILP가 schedule 결정) ↔ ② linalg P1 고정 패스(정책 C++ 하드코딩) ↔ ③ transform P2 반자동(사람이 schedule IR 명시). 이 study가 그 대조를 완성한다.
