학습 정점 — fusion 의 정수는 AffineMap 합성 (:44-71)

producer 입력의 새 indexing map 은 getIndexingMapOfProducerOperandsInCoordinatesOfFusedOp 가 argMap ∘ inv(producerResultMap) ∘ consumerArgMap 으로 만든다 — "producer 루프→arg 좌표" 를 result map 의 역으로 "텐서 좌표" 기준으로 바꾸고, consumer arg map 으로 "consumer(=fused) 루프" 기준까지 끌어온다. 합법성 검사의 permutation 조건(:129-132)이 바로 이 inversePermutation (:55-56) 의 존재 조건이다. body 병합은 generateFusedElementwiseOpRegion (:171-290) 이 두 payload 를 한 block 으로 splice 하고, linalg.index 는 consumerToProducerLoopsMap (:398-399 합성) 으로 remap. affine #0023 의 의존성 분석 기반 ad-hoc fusion 과 달리, 합법성 증명과 결과 계산이 같은 대수 연산 하나로 끝난다.

학습 정점 — ControlFusionFn: 합법성과 비용의 분리

using ControlFusionFn = std::function<bool(OpOperand*)> (Transforms.h:1650). areElementwiseOpsFusable 가 "해도 되는가" 를, controlFn 이 "할 것인가" 를 판정한다 (FuseElementwiseOps:428-431 에서 연속 호출). in-tree pass 의 defaultControlFn (:2139-2142) 은 producer->hasOneUse() 휴리스틱 — producer 복제(중복 계산)를 피한다. downstream 컴파일러(IREE 등)는 같은 populate 함수에 자체 cost model 을 꽂는다. 이식 시 가져갈 핵심 설계.

5 결론

  • 이식 검증 완료 — populate 4종(populateElementwiseOpsFusionPatterns, populateFoldReshapeOpsByExpansionPatterns, populateConstantFoldLinalgOperations, populateEraseUnusedOperandsAndResultsPatterns(간접)) + canonicalization 5그룹 + applyPatternsAndFoldGreedily(top-down) 를 같은 순서로 호출하면 in-tree 와 3/3 byte-identical. 알고리즘 재구현 0줄.
  • 개인 컴파일러 이식 가이드: 이 pass 자체는 deprecate 예정 (:2122-2127) 인 "패턴 시연용 합주" — 이식할 단위는 pass 가 아니라 (a) fuseElementwiseOps+areElementwiseOpsFusable 코어, (b) ControlFusionFn 자리에 꽂을 자체 cost model, (c) 필요 방향의 reshape 전파(expansion vs collapsing 선택) 세 가지다.
  • 구조 인사이트: 합법성(areElementwiseOpsFusable) / 비용(ControlFusionFn) / 변환(fuseElementwiseOps) 의 3분리. fusion 의 수학은 indexing map 의 대수적 합성 (argMap ∘ inv(resultMap) ∘ consumerMap) 하나로, producer result map 의 permutation 조건이 그 역함수 존재 조건. affine 의 의존성-분석 기반 fusion 과 정확히 대비되는 지점 (실험 README 의 baseline 비교 참고).
  • 후속 아이디어: (1) collapse 방향 (populateFoldReshapeOpsByCollapsingPatterns + populateCollapseDimensions) 을 별도 pass 로 묶어 expansion 과 왕복 관찰, (2) ControlFusionFn 을 Option<> 로 노출해 hasOneUse 휴리스틱 무력화 시 producer 복제가 실제로 일어나는지 관찰, (3) linalg.index 보유 generic 의 fusion 에서 index remap (:398-403) 확인.
