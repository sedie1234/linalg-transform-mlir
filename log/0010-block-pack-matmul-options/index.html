linalg-block-pack-matmul 옵션 LLVM/MLIR 19.1.7

linalg-block-pack-matmul의 옵션 8개 각각이 IR을 어떻게 바꾸는지, 옵션 하나만 토글한 실제 before/after로 보인다. 모든 MLIR은 my-mlir-opt --my-block-pack-matmul=...의 실제 출력이며(in-tree와 byte-identical), 각 변경 강조 토글로 바뀐 라인을 표시한다. 기준 입력은 linalg.matmul 64×256×128, 블록 (mb,nb,kb)=(32,16,64) (padding 계열만 30×30×30, 16).

① 기준 패킹 ② 옵션 한눈에 ③ block-factors ④ allow-padding ⑤ mnk-padded-multiples ⑥ mnk-order ⑦ transpose 4종 ⑧ no-op ⑨ 요약

① 기준 패킹 — 기본 옵션의 결과 (mmt4d 형)

먼저 기준이 되는 기본 패킹을 본다. linalg.matmul(64×256×128)에 block-factors=32,16,64만 주면, 2D 행렬곱이 tensor.pack ×3 + 6D linalg.generic + tensor.unpack으로 바뀐다. 이게 이후 모든 옵션 비교의 before 기준이다.

#map = affine_map<(d0, d1, d2, d3, d4, d5) -> (d0, d2, d3, d5)>
#map1 = affine_map<(d0, d1, d2, d3, d4, d5) -> (d1, d2, d4, d5)>
#map2 = affine_map<(d0, d1, d2, d3, d4, d5) -> (d0, d1, d3, d4)>
module {
  func.func @matmul_f32(%arg0: tensor<64x256xf32>, %arg1: tensor<256x128xf32>, %arg2: tensor<64x128xf32>) -> tensor<64x128xf32> {
    %0 = tensor.empty() : tensor<2x4x32x64xf32>
    %pack = tensor.pack %arg0 outer_dims_perm = [0, 1] inner_dims_pos = [0, 1] inner_tiles = [32, 64] into %0 : tensor<64x256xf32> -> tensor<2x4x32x64xf32>
    %1 = tensor.empty() : tensor<8x4x16x64xf32>
    %pack_0 = tensor.pack %arg1 outer_dims_perm = [1, 0] inner_dims_pos = [1, 0] inner_tiles = [16, 64] into %1 : tensor<256x128xf32> -> tensor<8x4x16x64xf32>
    %2 = tensor.empty() : tensor<2x8x32x16xf32>
    %pack_1 = tensor.pack %arg2 inner_dims_pos = [0, 1] inner_tiles = [32, 16] into %2 : tensor<64x128xf32> -> tensor<2x8x32x16xf32>
    %3 = linalg.generic {indexing_maps = [#map, #map1, #map2], iterator_types = ["parallel", "parallel", "reduction", "parallel", "parallel", "reduction"]} ins(%pack, %pack_0 : tensor<2x4x32x64xf32>, tensor<8x4x16x64xf32>) outs(%pack_1 : tensor<2x8x32x16xf32>) {
    ^bb0(%in: f32, %in_2: f32, %out: f32):
      %4 = arith.mulf %in, %in_2 : f32
      %5 = arith.addf %out, %4 : f32
      linalg.yield %5 : f32
    } -> tensor<2x8x32x16xf32>
    %unpack = tensor.unpack %3 inner_dims_pos = [0, 1] inner_tiles = [32, 16] into %arg2 : tensor<2x8x32x16xf32> -> tensor<64x128xf32>
    return %unpack : tensor<64x128xf32>
  }
}
6D 반복공간 (d0,d1,d2,d3,d4,d5) = (MB, NB, KB, mb, nb, kb), iterator [par, par, red, par, par, red]. 세 operand의 packed layout:
A (LHS)[M=64][K=256] → [MB=2][KB=4][mb=32][kb=64], map (d0,d2,d3,d5)
B (RHS)[K=256][N=128] → [NB=8][KB=4][nb=16][kb=64], map (d1,d2,d4,d5) — 기본 RHS transpose로 [NB][KB]… 순
C (out)[M=64][N=128] → [MB=2][NB=8][mb=32][nb=16], map (d0,d1,d3,d4)
접근 패턴이 정확히 linalg.mmt4d: [MB][NB][mb][nb] += [MB][KB][mb][kb] * [NB][KB][nb][kb].

아래 각 옵션 절은 이 기준에서 해당 옵션 하나만 토글한 결과다. "변경 강조"를 켜면 바뀐 라인만 색칠된다.

② 옵션 한눈에

옵션기본값한 줄 효과
block-factors (mb,nb,kb)— (필수)minor 블록 크기. 없으면 전체 no-op. 지정하면 pack×3+generic+unpack 생성
allow-paddingtrue안 나눠떨어질 때 padding_value로 패딩 허용. false면 full-tile 아니면 no-op
mnk-padded-multiples (M,N,K)—M·N·K 크기를 이 배수로 올림(ceil) → 더 크고 적은 블록 + 패딩
mnk-order (perm)0,1,2(M,N,K) 차원 순서 permutation → 블록 contraction의 루프/레이아웃 순서 변경
lhs-transpose-outer-blocksfalseLHS 바깥 블록 [MB][KB] → [KB][MB]
lhs-transpose-inner-blocksfalseLHS 안쪽 블록 [mb][kb] → [kb][mb]
rhs-transpose-outer-blockstrueRHS 바깥 블록 [KB][NB] → [NB][KB] (기본 mmt4d 형의 일부)
rhs-transpose-inner-blockstrueRHS 안쪽 블록 [kb][nb] → [nb][kb] (기본 mmt4d 형의 일부)
RHS 기본값 2개가 true라서 기본 결과가 mmt4d 형이 된다. 그래서 RHS 절의 before/after는 "기본(true) → 끔(false)" 방향으로 본다.

⑦ transpose 4종 — outer/inner 블록 배치 뒤집기

LHS·RHS 각각의 바깥(outer) 블록과 안쪽(inner) 블록 순서를 독립적으로 뒤집는다. 구현상 tensor.pack의 outer_dims_perm(outer) / inner_dims_pos·inner_tiles(inner)와 해당 indexing map을 같은 permutation으로 치환한다. LHS 기본은 둘 다 false, RHS 기본은 둘 다 true다.

⑧ no-op(변환 안 됨) 케이스

조건이유
block-factors 미지정모든 pattern이 "3개 tile factor 필요"로 matchFailure → IR 그대로
buffer semantics (memref 입력)hasPureBufferSemantics → "require tensor semantics" matchFailure. tensor 전용
allow-padding=false + 안 나눠떨어짐validateFullTilesOnDims 실패 → "packing full tiles only" matchFailure

모두 거절/에러가 아니라 match failure다 — greedy driver가 건너뛰고 입력을 그대로 둔다.

⑨ 요약

한 문단. block-factors가 변환의 필수 스위치이고(없으면 no-op), allow-padding·mnk-padded-multiples는 나눠떨어지지 않는 크기·타일 올림을 패딩으로 처리하며, mnk-order는 차원 순서를, transpose 4종(lhs/rhs-transpose-outer/inner-blocks)은 각 operand의 outer·inner 블록 배치를 조정한다. transpose 옵션은 tensor.pack의 outer_dims_perm/inner_dims_pos·tiles와 indexing map을 같은 permutation으로 바꾸는 것이고, LHS=false·false / RHS=true·true 기본 조합이 곧 mmt4d 레이아웃이다. 모든 IR은 LLVM/MLIR 19.1.7 my-mlir-opt 실제 출력(in-tree와 byte-identical)이다.
