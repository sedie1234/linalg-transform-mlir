linalg-fold-unit-extent-dims

크기 1(unit-extent) 차원을 linalg/tensor op에서 제거하는 정규화 pass. 사용 함수·옵션·입력별 동작·변환 결과·이점·도움 케이스를 인터랙티브하게 정리한다. 변환 전/후 MLIR은 변경 강조 토글로 바뀐 라인이 색칠된다. (LLVM 19.1.7, byte-identical 8/8.)

① 개요 ② 사용 함수 ③ 옵션 ④ 입력별 동작 ⑤ 이점 ⑥ 도움 케이스 ⑦ 요약

① 개요 — size-1 차원은 군더더기

unit-extent dim(크기 1)은 정보가 없는 군더더기다. 이 pass는 그것을 operand·iteration space에서 떼어 op을 정규형으로 만든다. 런타임 동작은 불변(데이터 이동 없음) — 가치는 그 자체가 아니라 fusion·vectorization·loop 제거 같은 진짜 최적화를 해금하는 데 있다. 핵심: tensor<1x5xf32> — collapse → tensor<5xf32> + indexing map 재작성.

아래에서 unit 차원이 어떻게 접히는지 — 데이터(셀)는 그대로, 논리 shape(축 라벨)만 단순화되는 모습을 재생해 보자. collapse_shape가 "메타데이터 변환"인 이유가 여기 있다.

③ 옵션 — use-rank-reducing-slices

유일한 옵션. unit 차원을 떼는 경계 op의 종류만 가른다 — 정규화 결과(map·rank)는 동일.

모드전략경계 op성격
defaultReassociativeReshapecollapse_shape / expand_shape메타데이터 뷰 (데이터 이동 0, reassociation 인접성 필요)
=trueExtractInsertSliceextract_slice / insert_slicestrided slice 의미론 (bufferization 친화성 다름)

같은 입력(@broadcast_add)에서 unit dim 제거 op이 모드에 따라 어떻게 바뀌는지 — 나머지 linalg.generic은 동일:

모드별 경계 op (그 외 동일)
default (reshape)
%collapsed = tensor.collapse_shape %arg0 [[0, 1]]
    : tensor<1x5xf32> into tensor<5xf32>
use-rank-reducing-slices=true
%extracted_slice = tensor.extract_slice %arg0[0, 0] [1, 5] [1, 1]
    : tensor<1x5xf32> to tensor<5xf32>
모드 비대칭: RankReduced{Extract,Insert}SliceOp는 default 모드에만 등록된다. 그래서 slices 모드에선 extract_slice/insert_slice 자체의 unit dim은 안 접히고, tensor.pad만 slice-기반 경계로 접힌다(④ pad-slice 참고). byte-identical 8/8은 "각 모드 안에서 본인 pass = in-tree"라는 뜻이지 두 모드 출력이 같다는 뜻이 아니다.

④ 입력별 동작 — 변환 전/후

실험(experiments/0007-*)의 4 입력. 각 변경 강조 토글로 collapse/expand·map 재작성 라인을 확인.

㉠ @broadcast_add — operand만 (broadcast 정리)

입력 두 operand가 unit dim으로 broadcast를 표현: arg0 1x5 map (0,d1)(d0축 broadcast), arg1 5x1 map (d0,0). 출력이 5x5라 iteration dim은 안 떨어지고, operand만 collapse + map의 상수0 자리 삭제.

변환 전후
before
%0 = linalg.generic {
    indexing_maps = [affine_map<(d0,d1)->(0,d1)>,
                     affine_map<(d0,d1)->(d0,0)>,
                     affine_map<(d0,d1)->(d0,d1)>],
    iterator_types = ["parallel","parallel"]}
  ins(%arg0, %arg1 : tensor<1x5xf32>, tensor<5x1xf32>)
  outs(%empty : tensor<5x5xf32>) {
^bb0(%a: f32, %b: f32, %c: f32):
  %add = arith.addf %a, %b : f32
  linalg.yield %add : f32
} -> tensor<5x5xf32>
after
%c0 = tensor.collapse_shape %arg0 [[0, 1]] : tensor<1x5xf32> into tensor<5xf32>
%c1 = tensor.collapse_shape %arg1 [[0, 1]] : tensor<5x1xf32> into tensor<5xf32>

    indexing_maps = [affine_map<(d0,d1)->(d1)>,

㉡ @drop_unit_loop_with_index — unit iteration dim + linalg.index

입력·출력이 1x8이고 d0=1이 one-trip loop. 루프가 떨어져 2D→1D, 결과는 expand_shape로 복원, linalg.index는 시프트(없어진 index 0 → const0, index 1 → index 0).

변환 전후
before
linalg.generic {maps=[(d0,d1)->(d0,d1), (d0,d1)->(d0,d1)],
                iterator_types=["parallel","parallel"]}
  ins(%arg0 : tensor<1x8xf32>) outs(%e : tensor<1x8xf32>) {
^bb0(%in: f32, %out: f32):
  %i = linalg.index 0 : index
  %j = linalg.index 1 : index
  %s = arith.addi %i, %j : index
  ... %add = arith.addf %in, %f : f32
  linalg.yield %add : f32
}
after
%c = tensor.collapse_shape %arg0 [[0, 1]] : tensor<1x8xf32> into tensor<8xf32>
%1 = linalg.generic {maps=[(d0)->(d0), (d0)->(d0)],
                     iterator_types=["parallel"]}        // 2D → 1D
  ins(%c : tensor<8xf32>) outs(%e8 : tensor<8xf32>) {
^bb0(%in: f32, %out: f32):
  %i = linalg.index 0 : index                            // index 1 → index 0 시프트
  ... %add = arith.addf %in, %f : f32
  linalg.yield %add : f32
} -> tensor<8xf32>
%e = tensor.expand_shape %1 [[0, 1]] output_shape [1, 8] : tensor<8xf32> into tensor<1x8xf32>

㉢ @unit_reduction — reduction → elementwise (MoveInitOperandsToInput) 큰 이득

1x?x1x1의 d0·d2·d3이 one-trip → 4-loop(par,par,red,red)가 1-loop(par)로. all-parallel이 되자 body가 %out(fill 결과)를 읽는 elementwise가 되어, MoveInitOperandsToInput이 fill을 ins로 이동. reduction op이 융합 가능한 elementwise로 변신.

변환 전후
before
%1 = linalg.fill ins(%cst : f32) outs(%0 : tensor<1x1xf32>) -> tensor<1x1xf32>
%2 = linalg.generic {
    indexing_maps = [(d0,d1,d2,d3)->(d0,d1,d2,d3), (d0,d1,d2,d3)->(d0,d2)],
    iterator_types = ["parallel","parallel","reduction","reduction"]}
  ins(%arg0 : tensor<1x?x1x1xf32>) outs(%1 : tensor<1x1xf32>) {
^bb0(%in: f32, %out: f32):
  %3 = arith.addf %in, %out : f32
  linalg.yield %3 : f32
} -> tensor<1x1xf32>
after
%c = tensor.collapse_shape %arg0 [[0,1,2,3]] : tensor<1x?x1x1xf32> into tensor<?xf32>
%1 = linalg.fill ins(%cst : f32) outs(%e0 : tensor<f32>) -> tensor<f32>   // rank-0
%3 = linalg.generic {maps=[(d0)->(d0), (d0)->(), (d0)->()],
                     iterator_types = ["parallel"]}                       // 4 → 1 loop
  ins(%c, %1 : tensor<?xf32>, tensor<f32>)                                // fill이 ins로!
  outs(%e1 : tensor<f32>) {
^bb0(%in: f32, %in_0: f32, %out: f32):
  %4 = arith.addf %in, %in_0 : f32
  linalg.yield %4 : f32
} -> tensor<f32>
%e = tensor.expand_shape %3 [] output_shape [1, 1] : tensor<f32> into tensor<1x1xf32>

㉣ @pad_unit — generic이 아닌 tensor op (tensor.pad)

이 pass는 generic을 넘어 tensor.pad·extract_slice·insert_slice의 unit dim도 정리한다. 1x16x1의 0·2번(size-1, pad 0)을 떼고 1D에서 pad 후 복원:

변환 전후 (default 모드)
before
%0 = tensor.pad %arg0 low[0, 2, 0] high[0, 2, 0] {
^bb0(%i: index, %j: index, %k: index):
  tensor.yield %cst : f32
} : tensor<1x16x1xf32> to tensor<1x20x1xf32>
after
%c = tensor.collapse_shape %arg0 [[0, 1, 2]] : tensor<1x16x1xf32> into tensor<16xf32>
%p = tensor.pad %c low[2] high[2] {
^bb0(%arg1: index):
  tensor.yield %cst : f32
} : tensor<16xf32> to tensor<20xf32>
%e = tensor.expand_shape %p [[0, 1, 2]] output_shape [1, 20, 1] : tensor<20xf32> into tensor<1x20x1xf32>
@slice_unit(extract_slice)·@insert_unit(insert_slice)도 default 모드에선 rank-reduced로 접히지만, slices 모드에선 불변(그 패턴이 미등록). pad만 slices 모드에서 extract_slice+insert_slice 경계로 접힌다 — ③의 모드 비대칭.

⑤ 이점 — 정규화가 최적화를 해금

런타임은 불변. 이득은 다른 최적화가 먹히게 만드는 것이다. 대표적으로 broadcast 정리가 fusion을 푸는 경로:

① 정규형

unit 차원만 다른 동일 연산이 같은 모양 → 패턴 매칭·CSE·dedup 개선.

② one-trip loop 제거

unit iteration dim = 1번 도는 루프 → drop. lowering 후 trivial 루프 오버헤드 소멸.

③ fusion 해금

map이 깨끗한 projection이 되고, reduction이 elementwise화(MoveInit) → producer-consumer 융합 대상.

④ vectorization

unit 차원은 width-1 벡터(lane 낭비) → 떼면 진짜 차원을 벡터화.

⑤ IR 축소

낮은 rank·적은 map 항목·적은 iteration → 모든 후속 패스 부담 감소.

비용

경계 reshape뿐(기본 메타데이터·복사 없음). 보통 canonicalize가 인접 reshape를 접어 소거.

⑥ 도움이 되는 케이스

상황이 pass가 하는 일해금되는 것
broadcast가 unit dim(1xN, Nx1)으로 인코딩됨unit dim collapse + map을 projection으로elementwise fusion
leading batch=1, channel=1 등 trivial 축그 축 droptiling·vectorization 단순화
reduction 차원이 전부 size-1elementwise화 + init를 ins로후속 generic과 융합
conv/pool 후 1-크기 spatial 잔존unit spatial dim collapselower-rank named op·loop 제거
pad/slice가 unit dim 포함낮은 rank에서 pad/slice 수행bufferization·연산 단순화
변환 파이프라인의 정규화 단계canonicalize·fusion 전에 unit dim 청소downstream 매칭 안정화

⑦ 요약

한 문단. linalg-fold-unit-extent-dims는 dropUnitDims(+ MoveInitOperandsToInput, DropPadUnitDims, RankReduced*SliceOp)로 크기 1 차원을 떼는 정규화다. operand는 collapse, unit iteration이면 loop drop + result expand, all-parallel이 된 reduction은 elementwise화. 옵션 use-rank-reducing-slices는 경계 op을 reshape(메타데이터) ↔ slice로만 가른다(결과 동일, 일부 패턴은 reshape 전용). 런타임 의미는 불변이고, 이득은 fusion·vectorization·one-trip loop 제거를 해금하는 데 있다 — "그 자체 최적화"가 아니라 "최적화를 잘 되게 하는 정규화".
